## Phase 1 UI Scope

This section documents the specific pages, components, and interactions required by Phase 1. These are derived from the Phase 1 success criteria and requirements DATA-01, DATA-02, DATA-07.

### Pages

| Page | Route | Purpose |
|------|-------|---------|
| Assets | `/assets` | List, add, edit, delete manual assets. Filter by category. |

### Layout Components

| Component | Description |
|-----------|-------------|
| AppLayout | Root layout with sidebar navigation and main content area |
| Sidebar | Left-side navigation with app title and route links |

### Page Components

| Component | Description |
|-----------|-------------|
| AssetList | Table/card list of all manual assets with columns: name, type, category, quantity, currency, institution |
| AssetForm | Create/edit form with fields: name, type (bank_current, bank_deposit, emergency_fund, stock, etf, etp), quantity, currency (EUR, USD, etc.), institution, macro category (Crypto, Stocks, Liquidity, Trading Bots) |
| CategoryFilter | Horizontal button group to filter assets by macro category (All, Crypto, Stocks, Liquidity, Trading Bots) |
| EmptyState | Displayed when no assets exist or no assets match the current filter |
| DeleteDialog | Confirmation dialog for asset deletion, uses destructive color for confirm button |

### Interaction Patterns

| Interaction | Behavior |
|-------------|----------|
| Add asset | Click "Add Asset" CTA -> modal or inline form with AssetForm -> on submit, asset appears in list |
| Edit asset | Click edit action on asset row -> form pre-populated with current values -> on submit, list updates |
| Delete asset | Click delete action on asset row -> DeleteDialog appears -> confirm -> asset removed from list |
| Filter by category | Click category filter button -> list filters to show only matching assets -> active filter uses accent color |
| Empty state | When no assets exist, show EmptyState component with guidance copy and "Add Asset" CTA |
| Error handling | On API error, display error state copy. On form validation error, show inline field-level messages. |

### Form Fields

| Field | Type | Validation | Notes |
|-------|------|------------|-------|
| Name | text input | required, min 1 char | Asset display name |
| Type | select dropdown | required | bank_current, bank_deposit, emergency_fund, stock, etf, etp |
| Macro Category | select dropdown | required | Crypto, Stocks, Liquidity, Trading Bots |
| Quantity | number input | required, > 0 | Decimal precision to 8 places |
| Currency | select dropdown | required, default EUR | EUR, USD, GBP, CHF, etc. |
| Institution | text input | optional | Bank or broker name |
