<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> API | Untrusted HTTP input crosses here (JSON body, query params, URL params) |
| API -> Database | Parameterized queries via TypeORM (trusted within app boundary) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Tampering | POST/PUT /api/assets body | mitigate | class-validator DTOs with @IsString, @IsEnum, @Length enforce type and range. ValidationPipe whitelist:true strips unknown fields. forbidNonWhitelisted:true rejects extra fields. |
| T-1-02 | Tampering | Asset name/notes fields (XSS vector) | mitigate | React auto-escapes rendered content on frontend. Backend stores as-is; NestJS response is JSON (no HTML rendering). Low risk in Phase 1. |
| T-1-03 | Tampering | SQL injection via query params | mitigate | TypeORM parameterized queries via createQueryBuilder with `:category` parameter binding. Never concatenate user input into SQL strings. |
| T-1-04 | Information Disclosure | Error responses leak internals | accept | NestJS default exception filter returns generic messages. Custom HttpExceptionFilter formats consistently. Single-user local app, low risk. |
</threat_model>
